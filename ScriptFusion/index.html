Just a quick breakdown for anyone asking "What about 98kickasskernel? Or "what about juwe RAM optimization?" "Don't they do the same thing?" In theory, sure... For RAM and sysctl values. Ever wonder why there are two versions of the 98kernel script to prevent bootloops? That is one key difference. This system doesn't just throw values at your device, or follow an "if you bootloop" methodology. Sure, things may cause loops because there is no reliable check (See odex/dalvikvm), but nothing is hard-coded.
"What about setCPU?" or "what about Advanced System Tools?" are answered much the same way. You can run five applications and use up a good chunk of the optimization they create running them together... or you can use one and keep the extra for yourself.
Another big difference is that those scripts cannot change screen density, tweak the CPU, change the scheduler, profile frequencies, and fix broken WiFi all in one run. Complete means complete, simple as that. Customization means user determined customization, not what one developer recommends. Recommendations will help through the process, but they remain exactly that. There are no rules. Have some fun...

I do not place any restriction on use of the script, components, or functions. All I ask is that the script is not used in the development of similar applications. Whether or not I am credited is at the discretion of the developer and their personal moral obligations. So what exactly makes this so different than any other tweak package / speedtweak?

Modified and rebuilt entirely on device. This means that updates happen just as soon as I run them. In the past, this slowed development due to unpacking and repacking images. With the current revision, that issue becomes one of its greatest strengths. Modified 6.2.1 tweak package built in. While launch scripts do make this method just as easy to launch for the end user, disabling features required manually editing system files, and there were quite a few to keep track of. I aim to minimize packed files, maximize ease of use, and eliminate manual edits.

-        Basic profile settings that will attempt to integrate available frequencies into a preset voltage profile list with min and max

-        Custom profile allowing the user to pick a minimum and maximum frequency value. The user will then be walked through setting voltage values for each available frequency (All frequencies are customized in case min or max are overridden).

-        Backup option allows the user to back up the frequency and voltage level file to either /data/data/ or /sdcard/ScriptFusion

-        Restore will take the backup file and place it back into the init.d and then launch it

-       Governor options that will read the available governors from the kernel and allow changing the current option.

-        Clear commands will clear the listed item. Available options include battery stats, dalvik-cache, and a combined clear.

-        Wifi Calibrate will clear the required wifi settings to restore operation when switching to a new ROM or kernel. Please note that this USUALLY fixes wifi (at the cost of having to reconnect to any previous saved networks) and prevents needing to revert the kernel or wipe data.

-        Fix permissions will run the same permission script that is run by virtually any fix permission function. It is a fairly generic script that has been working efficiently and did not need to be further modified

-        Wake on Volume keys will allow the toggling on and off of the wake on volume feature for volume up and volume down separately, based on their current status. If a key is set to wake, the option will be to disable. If the key is set not to wake, the option will be to enable.

-        Swap options allow the user to create and remove swap, with both immediate and ongoing results. There is no need to reboot when using these options, and all choices are persistent through system reboot.

-        Sheduler options to alter the scheduler based on a listing of available choices.  Extended options are offered for tweaking sleepers values, if the system supports changing those values.

-        Binary installers to upgrade / install busybox, modified wget, bash, etc.

-        Odex optimization (BETA) adds odex files to help increase cache speed. Backup first. May not work with Gingeritis due to dalvikvm.

-        Screen density editor will allow custom values to be entered for the screen "resolution" to change overall size appearance.

-        Updater-Script Editor will remove the data wipe functions from the META-INF folder of a ROM that has been extracted.

-        AdBlock options to enable, disable, and update the listing for the ads shown when using the browser and other applications.

-        Proximity Sensor options to enable and disable the proximity sensor that turns the screen on and off while phone is in use.

-        Tweak menu to toggle what tweaks are applied by category. They are grouped by how they are most often applied. For example, filesystem includes both remounts for partitions as well as sdcard read enhancement. By default, all tweaks are disabled to prevent issues caused by compatibility.

This list is still being updated, so many features are not included. Please check the thread, as most of the features not yet listed are explained throughout it.
